# Product Analytics Onboarding

База знаний для онбординга продуктовых аналитиков. Заметки написаны в Obsidian и хранятся в формате Markdown.

---

## Структура репозитория

```
├── AB.md          # A/B тестирование: методология, метрики, подводные камни
├── AI.md          # AI/ML в продуктовой аналитике
├── ML.md          # Машинное обучение: базовые концепции и применение
├── Managment.md   # Управление продуктом и работа с командой
├── Product.md     # Продуктовая аналитика: фреймворки, метрики, подходы
├── Python.md      # Python для аналитика: инструменты и библиотеки
├── SQL.md         # SQL: запросы, оптимизация, паттерны
├── Statistic.md   # Статистика: основы и применение в анализе данных
├── Теорвер.md     # Теория вероятностей
└── images/        # Изображения, используемые в заметках
```

## Как скачать и открыть в Obsidian

**1. Клонируй репозиторий в нужную папку**

```bash
git clone https://github.com/elesingp2/product-analytics-onboarding.git
```

**2. Открой папку как хранилище в Obsidian**

- Запусти Obsidian
- На стартовом экране выбери **"Open folder as vault"**
- Укажи папку `product-analytics-onboarding`

Все заметки и картинки подтянутся автоматически.

---

## Как стать контрибьютором

**1. Сделай форк репозитория** на GitHub (кнопка Fork вверху страницы)

**2. Клонируй свой форк**

```bash
git clone https://github.com/ТВО_ИМЯ/product-analytics-onboarding.git
```

**3. Создай ветку** с неймингом `имя_дата`:

```bash
git checkout -b ivan_2024-03-14
```

Формат: `имя` латиницей строчными буквами, затем `_`, затем дата в формате `YYYY-MM-DD`.

Примеры: `anna_2024-03-14`, `dmitry_2024-11-01`

**4. Внеси изменения, закоммить и запушь**

```bash
git add .
git commit -m "описание изменений"
git push origin ivan_2024-03-14
```

**5. Открой Pull Request** на GitHub из своей ветки в `main`

## Правила оформления заметок

- Картинки добавляй в папку `images/` с именами **без пробелов** (используй `_`)
- Ссылки на картинки в формате: `![](images/название_картинки.png)`
- Один файл = одна тема
