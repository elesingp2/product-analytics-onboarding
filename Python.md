#### Гитхаб

Гилхаб/гитлаб это мощный инструмент для версионирования.
Я хочу чтобы аналитик знал и умел пользоваться этими командами.

Задание: без ai протестировать эти команды на примере своего репозитория.

**git init** - инициализируем репозиторий
**git add .** - переход в стейдж (нужно чтобы гит понял различия между версиями)
**git commit -m 'name'** - коммитим
**git log** - посмотреть обновления
**git checkout 'хэш в логах'** - переход на данную версию
**git checkout HEAD~** - переход на прошлую версию программмы
**HEAD** - last commit
**git reset –hard(удалить) HEAD~** - удаляет сразу код и коммит
**git reset –soft(удалить) HEAD~** - не удаляет код, то есть можно комитнуть потом
**git checkout -b 'называние бранча'** - переход на другой бранч
**git diff newbranch** - разница бранчей
**git merge newbranch** - коммит новой ветки в мастер ветку
**git merge –no-commit –no-ff newbranch** - добавление новой ветки в мастер ветку без коммита (потом гит адд и гит коммит "merged branch")
**git push –set-upstream origin newbranch**
**git pull** `<repo>` - скачать обновления себе на комп или в кодспейс (не забываем чекаут на нужный бренч)
pull in github - аналог мерджа но с согласия техлида
git fetch - скачать себе на комп но не мерджить
git fork - копирует реп другого себе
